## 🚀 系统启动

### 方式一：一键启动（推荐）

```bash
python start_all.py
```

这会同时启动后端服务和前端界面。

### 方式二：分别启动

**步骤1：启动后端服务**

```bash
# 在终端1中执行
python backend/app.py
```

看到以下输出表示启动成功：
```
仿真环境已初始化
 * Running on http://localhost:5000
```

**步骤2：启动前端界面**

```bash
# 在终端2中执行
cd frontend/city-frontend
npm start
```

看到以下输出表示启动成功：
```
Compiled successfully!

You can now view city-frontend in the browser.

  Local:            http://localhost:3000
```

### 方式三：使用 VS Code 启动

1. 打开 VS Code
2. 按 `` Ctrl+` `` 打开终端
3. 在终端1执行：`python backend/app.py`
4. 点击右侧"+"号新建终端
5. 在终端2执行：
   ```bash
   cd frontend/city-frontend
   npm start
   ```

### 1️⃣ 控制面板（左侧）

**仿真控制按钮**
- **开始**：启动仿真，车辆开始移动
- **暂停**：暂停仿真，车辆停止移动
- **重置**：重置仿真到初始状态

**生成车辆**
- 下拉选择车辆类型：轿车、公交车、卡车、应急车
- 点击"生成车辆"按钮在地图上随机生成一辆车

**统计信息**
- 仿真时间：运行的时间（秒）
- 车辆数量：当前在地图上的车辆数
- 行人数量：当前在地图上的行人数
- 完成率：已完成任务的车辆百分比

### 2️⃣ 可视化画布（中央）

**显示内容**
- 灰色粗线：道路
- 白色虚线：车道分隔线
- 浅灰色圆圈：交叉口
- 彩色方块：车辆（不同颜色代表不同车型）
- 小圆点：交通信号灯（红/黄/绿）

**颜色说明**
| 颜色 | 含义 |
|------|------|
| 🔵 蓝色 | 轿车 |
| 🔴 红色 | 公交车 |
| 🟠 橙色 | 卡车 |
| 🟣 紫色 | 应急车 |
| 🟢 绿色 | 摩托车 |

**交互操作**
- **点击车辆**：选中该车辆，在右侧面板显示详情
- **观察交通流**：实时观看车辆移动和信号灯变化

### 3️⃣ 实体详情面板（右上）

**显示信息**
- 车辆ID
- 车辆类型（轿车/公交车/卡车）
- 当前状态（活跃/完成）
- 实时位置（X, Y坐标）
- 当前速度
- 最大速度
- 行驶时间

**获取决策按钮**
- 点击后系统会调用 LLM（大语言模型）
- 显示车辆的感知信息和决策结果
- 可以看到 AI 是如何决定加速、减速或停止的

### 4️⃣ 决策日志（右下）

**显示内容**
- 所有车辆的决策历史
- 时间戳
- 决策类型（加速/减速/停止）
- 决策原因

**用途**
- 分析交通流的决策模式
- 观察 AI 在不同情况下的决策

## 🎮 基础操作

### 操作1：启动仿真并观察

1. 打开浏览器访问 http://localhost:3000
2. 在左侧控制面板点击 **【开始】** 按钮
3. 观察中央画布，车辆开始移动
4. 注意交通信号灯会变色（红→黄→绿）

### 操作2：生成新车辆

1. 在左侧选择车辆类型（如"卡车"）
2. 点击 **【生成车辆】** 按钮
3. 在画布上观察新车辆出现
4. 车辆会自动规划路线并开始行驶

### 操作3：查看车辆详情

1. 在画布上 **点击任意车辆**
2. 右上方"实体详情面板"会显示该车辆信息
3. 可以看到车辆的ID、速度、位置等

### 操作4：获取 AI 决策

1. 先点击画布上的车辆选中它
2. 在右侧面板点击 **【获取决策】** 按钮
3. 等待几秒钟（LLM正在思考）
4. 下方会显示 AI 的决策输出：
   ```
   决策输出
   - 时间: 12.5s
   - 感知: {前方车辆距离: 30m, 信号灯: 绿色}
   - 决策: {action: "accelerate", reason: "前方畅通"}
   ```

### 操作5：暂停和重置

1. 点击 **【暂停】** 按钮，所有车辆停止移动
2. 点击 **【开始】** 按钮，车辆继续移动
3. 点击 **【重置】** 按钮，仿真回到初始状态

## 🧠 高级功能

### 功能1：配置 LLM（大语言模型）

系统支持使用 AI 大模型来做交通决策。

**步骤1：编辑配置文件**

打开 `config/siliconflow_config.json`：

```json
{
  "provider": "SILICONFLOW",
  "api_key": "sk-替换为你的真实API Key",
  "base_url": "https://api.siliconflow.cn/v1",
  "model": "Qwen/Qwen3-14B"
}
```

**步骤2：重启后端**

按 `Ctrl+C` 停止后端，然后重新运行：
```bash
python backend/app.py
```

**步骤3：测试 LLM**

1. 点击车辆
2. 点击"获取决策"
3. 观察决策结果（现在是由 AI 大模型生成的）

### 功能2：批量生成车辆

快速点击【生成车辆】按钮多次，可以模拟交通拥堵场景。

### 功能3：观察决策日志

1. 让仿真运行一段时间
2. 查看右下角的"决策日志"
3. 分析不同情况下的决策模式：
   - 绿灯时：通常是 "proceed"（继续行驶）
   - 前方有车时：通常是 "decelerate"（减速）
   - 红灯时：通常是 "stop"（停止）

## ❓ 故障排除

### 问题1：前端无法连接

**现象**：显示"未连接"

**解决**：
1. 确保后端已启动（`python backend/app.py`）
2. 刷新浏览器页面
3. 检查后端端口 5000 是否被占用

### 问题2：LLM 决策返回"服务不可用"

**现象**：决策显示"LLM服务不可用，使用默认策略"

**解决**：
1. 检查 `config/siliconflow_config.json` 中的 API Key 是否正确
2. 确保 API Key 没有过期
3. 检查网络连接是否能访问 api.siliconflow.cn

### 问题3：画布显示空白

**现象**：中央区域没有显示道路和车辆

**解决**：
1. 按 F12 打开浏览器开发者工具
2. 查看 Console 是否有错误信息
3. 刷新页面
4. 确保网络数据已加载（等待几秒钟）

### 问题4：npm 安装失败

**现象**：`npm install` 报错

**解决**：
```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 清理缓存后重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps
```

## 💡 使用技巧

1. **快速观察**：先点击【开始】，然后连续点击【生成车辆】几次，观察交通流

2. **AI 决策测试**：选中一辆车，多次点击【获取决策】，观察 AI 在不同情况下的决策变化

3. **拥堵模拟**：生成10辆以上车辆，观察它们如何排队和避让

4. **信号灯观察**：关注中央交叉口的信号灯变化，观察车辆如何响应
